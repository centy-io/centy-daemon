# Link System

## Overview

The link system provides relationship tracking between Centy items via per-link markdown files stored in `.centy/links/`. It covers the full lifecycle of link storage, retrieval, and display: links are stored as individual UUID-named markdown files with YAML frontmatter; the display-number resolution bug is fixed so same-numbered items of different types are correctly disambiguated; the legacy LinkTargetType protobuf enum is removed in favour of string-based item_type fields; and inline link data (with titles, directions, and counts) is surfaced directly in GetItem, ListItems, and all other item-retrieval APIs. This work makes links first-class, visible, and bidirectional across every item type in the project.

## Done

- #364 — Migrated link storage from per-entity JSON files to per-link markdown files at `.centy/links/{uuid}.md`; implemented direction field, removed inverse link types and `LinksFile` type, rewrote all CRUD functions.
- #367 — Removed the `LinkTargetType` protobuf enum and all associated Rust conversion helpers; migrated to string-based `*_item_type` fields throughout proto messages, Rust handlers, and e2e tests.

## Pending

- #360 (in-progress) — Fix display-number resolution scoped to item type so that e.g. `plan:1` and `issue:1` are not treated as the same entity, eliminating false SELF_LINK errors.
- #413 (open) — Surface link data inline in all item retrieval APIs: `GetItem` returns a populated `links` array; `ListItems`/`SearchItems` return per-item `link_count` with opt-in `--include-links`; all item types become valid link sources and targets.

## Implementation notes

Links are stored one-per-file in `.centy/links/` using mdstore; direction is synthesized at query time from the single stored record — no inverse files. Title resolution for `related_title` must use batch reads to avoid N+1 performance issues. Display-number lookup must always be scoped to item type to avoid cross-type collisions. The LinkTargetType removal follows the same pattern as the earlier EntityType enum removal (commit 0479d56).
