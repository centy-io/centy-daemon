# Rust Lint & Code Quality Hardening

## Overview

Comprehensive effort to enforce strict Clippy linting, remove all `#[allow]` suppression directives, and integrate custom Dylan linting rules across the Centy daemon codebase. This initiative hardened the codebase against panics, unsafe indexing, arithmetic overflow, and style violations through 113 discrete tasks — all now complete.

## Done

- Configured Clippy and pre-push hooks; set up CI lint enforcement
- Enabled strict safety lints: `indexing_slicing`, `string_slice`, `arithmetic_side_effects`, `expect_used`, `unwrap_used`
- Enabled code quality lints: `match_same_arms`, `map_unwrap_or`, `manual_let_else`, `unused_async`, `if_then_some_else_none`, `or_fun_call`, `shadow_same`, `option_if_let_else`, `useless_let_if_seq`, and 20+ more zero-violation lints
- Integrated custom Dylan linter for max-lines-per-function and max-lines-per-file enforcement
- Systematically removed all `#[allow]` suppression directives across the entire codebase (75 files)
- Split oversized files and handlers into modules to satisfy file/function size limits
- Removed deprecated `match_on_vec_items` lint and consolidated redundant lint entries in Cargo.toml
- Fixed all Dylan violations: long files, long functions, unnecessary async, wildcard imports, expect calls
- Remove unnecessary #[allow(dead_code)] from config/mod.rs (#307)
- Remove #[allow(unnecessary_async)] from src/server/handlers/init/handler.rs (#312)
- fix: remove async from assert_initialized — function never awaits (#313)
- fix: remove unnecessary async from assert_both_initialized helpers (#314)
- fix: deny field_scoped_visibility_modifiers and remove pub(super) from struct fields (#350)
- fix: deny option_if_let_else and fix 5 violations (#351)
- fix: deny useless_let_if_seq and fix violation in config/io.rs (#352)
- Always sort cspell words list alphabetically (a-z) (#357)
- fix: deny shadow_same clippy lint and fix 2 violations (#359)

## Implementation notes

All lints are now enforced at the Cargo.toml level with `deny` directives. The pre-push hook runs `cargo clippy -- -D warnings` and the Dylan custom linter before allowing pushes. No `#[allow]` suppressions remain except one intentional `wildcard_imports` allow in `trait_impl.rs` (required by the Dylan lint itself).
