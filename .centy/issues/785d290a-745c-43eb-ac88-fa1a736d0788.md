# Replace custom frontmatter parsing with a library

We have a hand-rolled frontmatter parser/generator in `src/common/frontmatter.rs` (~175 lines + tests). This should be replaced with an established crate to reduce custom code and maintenance burden.

## Current state

* `parse_frontmatter<T>` and `parse_frontmatter_raw` — split `---` delimited YAML, extract title and body
* `generate_frontmatter<T>` and `generate_frontmatter_raw` — serialize back to markdown with frontmatter
* Custom `FrontmatterError` type
* Used across issues, docs, generic storage, reconciliation, and tests

## Scope

* Pick a Rust frontmatter crate (e.g. `gray_matter` or `frontmatter`)
* Replace all parse/generate functions with library calls
* Remove `src/common/frontmatter.rs` entirely
* Update all call sites and tests
* The title/body splitting convention (H1 heading after frontmatter) may need a thin wrapper on top of the library
