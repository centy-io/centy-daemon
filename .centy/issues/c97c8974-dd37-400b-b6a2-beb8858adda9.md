# Auto-generate CLI from proto/gRPC definitions

## Background

The MCP server tools are fully auto-generated from the proto file via `protoc-gen-go-mcp`, producing ~11,500 lines of boilerplate in `mcp/gen/centy/v1/centy.pb.mcp.go`. Every RPC becomes an MCP tool automatically with `buf generate`.

The CLI has no equivalent — there are no per-RPC subcommands generated from proto. Any client CLI today must be hand-written.

## Goal

Add a code generation stage (similar to the MCP pipeline) that produces a CLI with subcommands for each RPC (e.g. `centy item create`, `centy project list`), with flags mapped from request message fields.

- `protoc-gen-cobra` — a `protoc` plugin that generates [Cobra](https://github.com/spf13/cobra) CLI commands from proto service definitions. Would slot in as a third plugin in `mcp/buf.gen.yaml` alongside `protoc-gen-go` and `protoc-gen-connect-go`. Zero hand-written subcommand boilerplate.

## Caveats

- `protoc-gen-cobra` maturity should be evaluated against the proto message complexity here (nested messages, enums, repeated fields)
- Generated CLI and generated MCP would share the same ConnectRPC client, keeping them in sync automatically as the proto evolves
