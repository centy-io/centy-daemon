# Release workflow: include CLI binary in release artifacts

The `cli/` directory contains a Go application auto-generated from proto/gRPC definitions (added in commit 56008ce), but the release workflow does not build or publish CLI binaries.

## What's missing

The `release.yml` workflow builds and distributes:
- `centy-daemon` (Rust) — platform-specific installers (.deb, .rpm, .pkg, .msi)
- `centy-mcp` (Go) — precompiled binaries + npm package

But `centy-cli` (Go) is not built or uploaded as part of the release.

## Proposed solution

Add a `build-cli` job to `release.yml` (similar to `build-mcp`) that compiles `centy-cli` for all 5 supported platforms:
- `linux-x86_64`
- `linux-aarch64`
- `darwin-x86_64`
- `darwin-aarch64`
- `windows-x86_64`

And uploads the binaries to the GitHub release. Optionally, publish a `centy-cli` npm package (similar to `centy-mcp`) for easy installation.
