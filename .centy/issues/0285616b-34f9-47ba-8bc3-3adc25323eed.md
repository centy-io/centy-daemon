# Thin out trait_impl.rs — split 95+ gRPC delegates by domain

## Problem

`src/server/trait_impl.rs` (422 lines) implements the entire gRPC service trait as a single file with 95+ pass-through delegate methods. This should be a thin routing layer, not a monolith.

## Proposal

Group handler delegates by domain and move them into domain-specific modules:
- `trait_impl/items.rs`
- `trait_impl/links.rs`
- `trait_impl/projects.rs`
- `trait_impl/users.rs`
- `trait_impl/workspaces.rs`
- etc.

Keep `trait_impl.rs` as a thin top-level trait impl that delegates to these submodules.
