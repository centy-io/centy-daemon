# Workspace & editor integration improvements

Consolidates several related workspace creation and editor integration improvements into a single tracking issue.

## Checklist

- [ ] **Unified `OpenTempWorkspace` RPC** (was #134): Merge `OpenInTempVscode` and `OpenInTempTerminal` into a single `OpenTempWorkspace` RPC that accepts an `editor_type` parameter (`EditorType` enum: `VSCODE`, `TERMINAL`). Add `OpenTempWorkspaceWithEditorRequest` proto message, implement unified handler in `server/mod.rs`, and deprecate the old RPCs.

- [ ] **Fix `plan.md` placement** (was #80): When opening a workspace with VSCode, `plan.md` is incorrectly written to `{workspace_path}/plan.md`. It should be placed at `.centy/issues/{issue_id}/plan.md`, alongside `issue.md`. Fix the path in `src/workspace/create.rs`.

- [ ] **Standalone workspaces** (was #115): Support creating temporary workspaces not tied to any issue (for ad-hoc exploration, prototyping, AI-assisted work). Add `OpenStandaloneWorkspaceRequest`/`Response` proto messages and a new `OpenStandaloneWorkspace` RPC. Make `issue_id` optional in workspace metadata, update `ListTempWorkspaces` to flag standalone vs issue-based workspaces, update naming logic, and provide a generic prompt template for standalone workspaces.

- [ ] **Reopen existing workspace** (was #100 and #96): When a workspace folder already exists (e.g. user opens the same issue twice, or a previous workspace wasn't cleaned up), instead of showing a fatal error, prompt the user with two options:
  - **Open existing** — focus the already-open editor window for this workspace.
  - **Delete and recreate** — remove the existing folder/worktree and create a fresh one.
  Implement across the app, CLI, and TUI. The error originates in `src/pr/git.rs` `create_worktree()`; check path existence before creating and return a structured response with options.

- [ ] **Unlink new workspace from existing VSCode workspace** (was #91): When creating a new workspace with the VSCode IDE, the new folder is incorrectly linked into the user's existing workspace. New workspaces should open as independent, unlinked VSCode windows.

## Affected Files

- `src/workspace/create.rs` — plan.md path, workspace creation logic
- `src/pr/git.rs` — `create_worktree()` error handling
- `server/mod.rs` — unified RPC handler
- Proto definitions — new messages and RPCs
- App, CLI, and TUI — reopen/recreate prompt UX
