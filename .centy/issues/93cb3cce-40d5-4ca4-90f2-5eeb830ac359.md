# Add epic as a default item type (statusless)

\`epic\` should be included as a default built-in item type alongside \`issues\`, \`stories\`, etc. Unlike other item types, epics are statusless — they do not have a \`status\` field and no status-related logic applies to them.

## Requirements

- \`epic\` is available as a default item type without any project-level configuration.
- The epic item type has no \`status\` field (statusless).
- Status-related operations (e.g. filtering by status, status transitions, status color mapping) must gracefully skip or ignore epic items.
- All existing item type functionality (create, read, update, delete, link, list) works for epics.

## Notes

- Statusless means the \`status\` field is absent from the frontmatter, not set to a null/empty value.
- The \`workspace.updateStatusOnOpen\` config option must not apply to epics.
