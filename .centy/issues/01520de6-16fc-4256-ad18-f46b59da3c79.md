# Remove PR entity type

Remove the entire PR (pull request) subsystem from the daemon. PRs will not be a built-in entity type. With the config.yaml approach (#135), the init command creates only issue and doc types. If users need a PR-like type, they can create .centy/prs/config.yaml as a custom item type.

## Source Code & Handlers

* Delete src/item/entities/pr/ directory entirely
* Delete src/server/handlers/pr\_\*.rs files (pr_create, pr_read, pr_list, pr_update, pr_delete, pr_soft_delete, pr_restore, pr_by_uuid)
* Remove PR-related exports from src/lib.rs
* Remove PR handler registrations from src/server/ (router, mod declarations)
* Remove HookItemType::Pr variant from hooks config
* Remove PullRequest variant from ItemType enum
* Fix all compilation errors from removals

## Proto Definitions & Message Types

* Remove PR RPC methods from proto/centy/v1/centy.proto (CreatePr, GetPr, GetPrByDisplayNumber, GetPrsByUuid, ListPrs, UpdatePr, DeletePr, SoftDeletePr, RestorePr, GetNextPrNumber)
* Remove PR-related message types (requests, responses, PrMetadata, PrWithProject)
* Regenerate proto stubs
* Update server trait implementation to remove PR RPCs

## Tests & E2E Specs

* Delete tests/pr_test.rs
* Delete e2e/prs.e2e.spec.ts
* Remove any PR references in shared test utilities or fixtures
* Verify remaining tests still pass

## Init & Data References

* Remove .centy/prs/ directory creation from init command
* Remove PR-related default config (statuses, etc.) from init
* Remove PR references from reconciliation/managed files
* Remove any PR folder scanning from project loading
* Update CLI commands that reference PR (centy create pr, centy list prs, etc.)
