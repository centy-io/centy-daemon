# Support custom item types and templates

Enable developers to define custom item types (epics, tasks, milestones) with fully customizable fields and templates.

## Goal

Allow projects to configure custom item types beyond the built-in issues/docs, with:

* Fully customizable fields per type
* Custom templates for each type
* Reference fields to link between types

## Example Use Cases

* **Epics** - high-level items grouping multiple issues
* **Tasks** - smaller work items, possibly nested under issues
* **Milestones** - time-based groupings with deadlines

## Implementation Steps

1. Config schema for custom types in config.json
1. Generic storage layer for custom type items
1. Generic CRUD operations
1. Field validation
1. Custom templates per type
1. gRPC API updates
