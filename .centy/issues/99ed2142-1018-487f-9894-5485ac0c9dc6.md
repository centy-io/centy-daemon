# Advanced issue searching with pagination and org filtering

## Overview

Implement a comprehensive advanced search feature for issues, covering the core search engine, result pagination, and organization-scoped filtering.

## Background

This consolidates three related issues:
- **#44** — Advance issue searching (parent feature)
- **#62** — Add pagination support for advanced search
- **#63** — Add organization filtering for advanced search

## Requirements

### Core Advanced Search
- [ ] Design and implement a query language / filter DSL for issue search
- [ ] Support filtering by status, priority, tags, and date ranges
- [ ] Support full-text search across issue title and body
- [ ] Support sorting results by field (e.g. priority, created_at, updated_at)
- [ ] Expose an `AdvancedSearch` RPC endpoint (request/response proto messages)
- [ ] Return structured `SearchResultIssue` objects including matched fields

### Pagination
- [ ] Add `page` / `page_size` (or `cursor`-based) pagination parameters to the search request
- [ ] Return total result count and next-page cursor/offset in the response
- [ ] Enforce a sensible default and maximum page size
- [ ] Ensure consistent ordering across pages

### Organization Filtering
- [ ] Allow scoping a search query to a specific organization
- [ ] Support filtering across all projects within an org in a single request
- [ ] Validate that the caller has access to the requested organization
- [ ] Return the originating project path alongside each result when searching org-wide

## Acceptance Criteria

- A single `AdvancedSearch` RPC accepts query, pagination, and org-filter parameters
- Results are correctly paginated and stable across pages
- Searching within an org returns issues from all member projects
- Searching without an org filter behaves as before (single-project scope)
- Unit and integration tests cover query parsing, pagination boundaries, and org scoping
