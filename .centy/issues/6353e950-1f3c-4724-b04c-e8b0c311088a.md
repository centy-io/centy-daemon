# Add cross-project ListItems RPC for global latest issue queries

## Summary

Add a new `ListItemsAcrossProjects` RPC (or extend `ListItems`) to support querying items across all tracked projects, sorted by `created_at` descending.

## Requirements

- New RPC that iterates over all tracked projects and aggregates items of a given type
- Support `limit` and `offset` for pagination
- Support sorting by `created_at` (descending by default) so callers can get the globally latest item
- Return `ItemWithProject` entries (like `SearchItems`) so the source project is included
- Support the same filters as `ListItems` (status, priority, MQL filter)

## Motivation

Users want to query the most recently created issue across all projects via the CLI (e.g., `centy list issues --global --limit 1`). The daemon needs to support this aggregation.

## After implementation

Publish a new daemon version.
