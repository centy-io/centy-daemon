# Add migration to insert config.yaml into existing projects

Parent: #135

Existing projects already have .centy/issues/ and .centy/docs/ folders but no config.yaml files. Add a migration that creates these files on centy update.

## Scope

* Detect projects missing config.yaml in issues/ and docs/ directories
* Generate default config.yaml for each, preserving existing project settings from config.json (allowedStates, priorityLevels, etc.)
* Map existing config.json fields to new config.yaml fields:
  * allowedStates -> statuses
  * defaultState -> defaultStatus
  * priorityLevels -> priorityLevels
  * customFields -> customFields
* Add migration version check so it only runs once
* Integration test with a sample pre-migration project directory
