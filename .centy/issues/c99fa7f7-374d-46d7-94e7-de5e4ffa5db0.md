# Add common trait interface for Issues and Docs

## Overview

Issues, PRs, and Docs share similar capabilities but currently lack a unified trait interface. This issue tracks the work to define and implement shared traits (`Timestamped`, `SoftDeletable`, `OrgSyncable`) across all entity types, and to consolidate duplicated fields into `CommonMetadata`.

### Current State

| Field | Issue | PR | Doc |
|---|---|---|---|
| `created_at` | via `CommonMetadata` | via `CommonMetadata` | direct |
| `updated_at` | via `CommonMetadata` | via `CommonMetadata` | direct |
| `deleted_at` | direct | direct | direct |
| `display_number` | via `CommonMetadata` | via `CommonMetadata` | none |
| `status` | via `CommonMetadata` | via `CommonMetadata` | none |
| `priority` | via `CommonMetadata` | via `CommonMetadata` | none |
| `is_org_X` | yes | no | yes |
| `org_slug` | yes | no | yes |

### Problems

1. `CommonMetadata` is shared by Issue/PR but NOT by Doc
2. `deleted_at` is duplicated in all three types rather than centralized in `CommonMetadata`
3. No unified trait interface for common operations
4. `OrgSyncable` trait exists but only Issues implement it — Docs do manual sync with duplicated logic

### Context

This is Phase 1a of the item-agnostic migration (#164). The trait interfaces built here become the foundation for the config.yaml-driven item type system (#135).

## Checklist

### Trait definitions
- [ ] Create `src/common/traits.rs` with `Timestamped` and `SoftDeletable` traits:
  ```rust
  pub trait Timestamped {
      fn created_at(&self) -> &str;
      fn updated_at(&self) -> &str;
      fn touch(&mut self);
  }

  pub trait SoftDeletable: Timestamped {
      fn deleted_at(&self) -> Option<&str>;
      fn is_deleted(&self) -> bool { self.deleted_at().is_some() }
      fn soft_delete(&mut self);
      fn restore(&mut self);
  }
  ```
- [ ] Export both traits from `src/common/mod.rs`
- [ ] Add unit tests for the `is_deleted()` default method
