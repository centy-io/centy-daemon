# Complete GetItem feature parity with legacy Get RPCs

Parent: #179
Supersedes: #151, #152

## Current State

The generic `GetItem` RPC exists (`proto/centy/v1/generic_item.proto:50-54`, handler in `src/server/handlers/item_read.rs`) and supports get by UUID or slug within a single project. However, it still lacks feature parity with the old per-entity Get RPCs.

## Gap 1 — display_number support

The old `GetIssueByDisplayNumber` and `GetPrByDisplayNumber` RPCs have no generic equivalent.

* Add `optional uint32 display_number = 4` to `GetItemRequest`
* Handler checks if set and dispatches to a new `generic_get_by_display_number()` in the storage layer
* Validate that the requested item type supports display numbers (issues, PRs — not docs)

## Gap 2 — cross-project search

The old `GetIssuesByUuid` and `GetDocsBySlug` RPCs search across all projects. The generic `GetItem` only works within a single project.

* Add new `SearchItems` RPC with `SearchItemsRequest`, `ItemWithProject`, and `SearchItemsResponse` messages
* New handler in `src/server/handlers/item_search.rs` following the pattern from `issue_by_uuid.rs` / `doc_read.rs`
* New storage function `generic_search_across_projects()`

## Gap 3 — deprecate legacy Get RPCs

Once parity is achieved, add deprecation comments to the old per-entity RPCs:

* `GetIssue`, `GetIssueByDisplayNumber`, `GetIssuesByUuid`
* `GetDoc`, `GetDocsBySlug`
