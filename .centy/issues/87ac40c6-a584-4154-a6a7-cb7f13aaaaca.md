# Replace custom gRPC logging Tower layer with tower-http TraceLayer

## Context

\`src/grpc_logging.rs\` (~96 lines) implements a custom Tower \`Layer\`/\`Service\` that logs gRPC request/response timing. This is a full manual Tower middleware implementation.

## Problem

\`tower-http\` is already a dependency. Its \`TraceLayer\` provides this exact functionality — request tracing with timing, method/path extraction, and response logging — without custom middleware code.

## Proposed Change

1. Add \`trace\` feature to \`tower-http\` in \`Cargo.toml\`
2. Replace \`GrpcLoggingLayer\` / \`GrpcLoggingService\` in \`src/grpc_logging.rs\` with \`tower_http::trace::TraceLayer\` configured with appropriate on-request/on-response callbacks
3. Delete or simplify \`src/grpc_logging.rs\`

## Files

- \`src/grpc_logging.rs\` — custom Tower middleware to remove
- \`src/main.rs\` — where the layer is applied to the server
