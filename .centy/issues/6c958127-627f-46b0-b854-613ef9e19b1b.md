# MCP: corrupted npx cache prevents centy-mcp from starting

When Claude Code tries to connect to the `centy` MCP server via `npx centy-mcp`, it fails with:

```
npm error code ENOTEMPTY
npm error syscall rename
npm error path /Users/ofek/.npm-cache/_npx/15e83008e763417e/node_modules/centy-mcp
npm error dest /Users/ofek/.npm-cache/_npx/15e83008e763417e/node_modules/.centy-mcp-bdvAiXzB
npm error errno -66
npm error ENOTEMPTY: directory not empty, rename ...
```

## Root cause

The npx cache entry for centy-mcp gets into a corrupted/incomplete state, causing npm to fail when trying to update it.

## Fix

Delete the corrupted cache entry:
```bash
rm -rf ~/.npm-cache/_npx/15e83008e763417e
```
