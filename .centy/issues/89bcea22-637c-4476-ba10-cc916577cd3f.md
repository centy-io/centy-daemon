# feat: comments as a built-in item type

## Summary

Add comments to centy by treating them as a built-in item type called \`comments\`. This reuses all existing generic item infrastructure (proto RPCs, storage, handlers) without any new modules or RPCs.

## Storage Layout

\`\`\`
.centy/
  comments/
    config.yaml        ← auto-created by init (like issues/config.yaml)
    <uuid>.md          ← one file per comment (UUID identifier)
\`\`\`

Each comment file uses existing mdstore frontmatter with custom fields:
- \`item_id\` — UUID of the parent item
- \`item_type\` — e.g. \`"issues"\`, \`"docs"\`
- \`author\` — free-form string

## Changes Required

1. **\`src/config/item_type_config/defaults.rs\`** — Add \`default_comment_config()\`
   - \`identifier: IdStrategy::Uuid\`
   - All features disabled (no display_number, priority, soft_delete, assets, org_sync, move, duplicate)
   - Custom field definitions for \`item_id\`, \`item_type\`, \`author\`

2. **\`src/reconciliation/managed_files.rs\`** — Add \`comments/\` directory to \`get_managed_files()\`

3. **\`src/reconciliation/migrate.rs\`** — Auto-create \`comments/config.yaml\` in \`migrate_to_item_type_configs()\` (same pattern as issues/docs/archived)

4. **\`src/config/item_type_config/mod.rs\`** — Re-export \`default_comment_config\`

5. **\`src/server/handlers/item_list/filters.rs\`** — Extend MQL filter to support \`{"customFields": {"item_id": "..."}}\` for server-side filtering by parent item

## Usage (no new RPCs needed)

All existing generic item RPCs work with \`item_type: "comments"\`:

| Operation | RPC | Custom fields |
|-----------|-----|---------------|
| Add comment | \`CreateItem\` | \`item_id\`, \`item_type\`, \`author\` |
| List by item | \`ListItems\` | filter: \`{"customFields": {"item_id": "<id>"}}\` |
| Edit comment | \`UpdateItem\` | update \`body\` |
| Delete comment | \`DeleteItem\` | \`force: true\` |

## Zero new files, zero new proto RPCs.
