# Remove llm-work module and all LLM/agent management code

## Problem

The centy-daemon currently contains an `llm` module (`src/llm/`) that handles LLM agent spawning, work session tracking, and prompt generation. This daemon should not be responsible for creating or managing agents or LLMs - that responsibility belongs elsewhere.

## Files to Remove

The entire `src/llm/` module:

* `src/llm/mod.rs` - Module exports
* `src/llm/work.rs` - Work session tracking (`llm-work.json`)
* `src/llm/agent.rs` - Agent spawning logic
* `src/llm/prompt.rs` - Prompt building
* `src/llm/config.rs` - LLM configuration
* `src/llm/generate.rs` - Title generation

## References to Update

Files that import from the `llm` module:

* `src/lib.rs`
* `src/server/mod.rs`
* `src/item/entities/issue/create.rs`
* `src/workspace/mod.rs`

## Additional Cleanup

* Remove `llm_work()` path from the proposed `ProjectPaths` struct (issue #81)
* Remove any `llm-work.json` file handling
* Clean up any related dependencies in `Cargo.toml`

## Rationale

The daemon should focus on project/issue management, not LLM orchestration. Keeping concerns separated makes the codebase more maintainable and allows the LLM management to be handled by the appropriate service.
