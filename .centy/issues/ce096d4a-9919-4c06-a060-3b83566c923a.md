# Replace gRPC ItemType enum with string item_type field

Parent: #130

Update the proto definitions to use string item_type instead of the ItemType enum.

## Scope

* Add string item_type field to CreateItemRequest, GetItemRequest, ListItemsRequest, UpdateItemRequest, DeleteItemRequest
* Mark existing ItemType enum as deprecated
* Server validates item_type against the registry, returns Error::UnknownType if not found
* Update existing handlers to read item_type string
* Backward compatibility: accept both enum (for v1 clients) and string for transition period
* Update proto tests
