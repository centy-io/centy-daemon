# Achieve 100% test coverage and enforce it in the pre-push hook

## Background

Currently `cargo test` passes 7 tests. There are 549 `#[test]`/`#[cfg(test)]` occurrences across 133 files covering specific logic (parsing, serialization, validation, reconciliation), but large portions of the server handlers, registry operations, and gRPC layer have no tests. No coverage tooling is configured. Both `cargo-llvm-cov` and `cargo-tarpaulin` are already installed.

## Plan

### 1. Measure current coverage

```sh
CENTY_HOME="$(mktemp -d)" cargo llvm-cov --all-targets --lcov --output-path lcov.info
```

Inspect the report to find uncovered modules. Priority areas:
- `src/server/handlers/` — gRPC request handlers
- `src/registry/` — project/org/user registry operations
- `src/reconciliation/` — plan building and execution edge cases
- `src/item/` — create/update/delete/move edge cases

### 2. Write tests to cover gaps

Follow the existing pattern: co-located `*_tests.rs` files or inline `#[cfg(test)]` modules. Use the `CENTY_HOME` env-var isolation pattern for any tests touching the registry or filesystem.

### 3. Add coverage gate to the pre-push hook

In `.cargo-husky/hooks/pre-push`, add after the existing test step:

```sh
echo "[N/N] Checking test coverage..."
CENTY_HOME="$(mktemp -d)" cargo llvm-cov --all-targets --fail-under-lines 100
```

## Acceptance criteria

- `cargo llvm-cov --all-targets --fail-under-lines 100` exits 0
- The pre-push hook includes this check and blocks pushes when coverage drops below 100%
