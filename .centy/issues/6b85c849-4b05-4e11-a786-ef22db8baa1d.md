# Add gRPC health checks to the daemon

Implement standard gRPC health checking for the daemon using the official gRPC Health Checking Protocol (`grpc.health.v1`).

## Goal

Expose health status via the standard gRPC health service so any gRPC-aware client (e.g. `grpc_health_probe`, load balancers, orchestrators) can query daemon liveness/readiness without any custom logic in the CLI or MCP.

## Requirements

- Implement the `grpc.health.v1.Health` service (`Check` and `Watch` RPCs) on the daemon's gRPC server
- Register the health server alongside existing services at startup
- Set service status to `SERVING` once the daemon is ready, and `NOT_SERVING` / `SERVICE_UNKNOWN` on shutdown or degraded state
- No custom health-check code in the CLI or MCP — consumers should use standard gRPC health tooling

## References

- gRPC Health Checking Protocol: https://github.com/grpc/grpc/blob/master/doc/health-checking.md
- `google.golang.org/grpc/health` package for Go implementations
