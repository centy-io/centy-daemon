# link: display numbers resolved globally across types causing false SELF_LINK

## Description

When two items of different types share the same display number (e.g., `issue #1` and `plan #1`), the daemon resolves them to the same entity when looking up by display number. This causes a false `SELF_LINK` error when trying to link them.

## Steps to Reproduce

1. Create an issue — it gets `displayNumber: 1`
2. Create a plan — it also gets `displayNumber: 1`
3. Attempt to link them:
   ```
   centy link plan 1 relates-to issue:1
   ```
4. Error: `Cannot link entity to itself` (`SELF_LINK`)

## Expected Behavior

Display number lookup should be scoped to the item type. `plan:1` and `issue:1` are different entities and should be linkable.

## Actual Behavior

```json
{"message":"Cannot link entity to itself","tip":"Cannot link an item to itself","code":"SELF_LINK"}
```
