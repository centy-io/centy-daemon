# Unify path construction with ProjectPaths struct and enforce unique project names

## Goal

Establish a single source of truth for all path construction in the codebase, enforce unique project names within an organization, and improve workspace folder naming to include the project name.

These are interconnected concerns: the `ProjectPaths` struct centralizes how paths are built, unique project name enforcement is required for unambiguous URL routing in `centy-app`, and workspace folder naming depends on having a reliable project name available.

---

## Background

### Path construction (replaces #27, #81)

Paths are currently constructed ad-hoc throughout ~15 modules using repeated `.join()` calls (e.g., `get_centy_path(project_path).join("issues").join(issue_id).join("issue.md")`). This is duplicated, error-prone, and hard to maintain.

### Unique project names (replaces #82)

When assigning a project to an organization, there is no validation preventing two projects from sharing the same directory name. This causes ambiguous URLs in `centy-app` (e.g., `/my-org/myapp/issues` would be ambiguous if two projects are both named `myapp`).

### Workspace folder naming (replaces #83)

Workspace folders are currently named `centy-{short_issue_id}-{timestamp}`, making it hard for users to identify which project is open in the IDE — especially when juggling multiple workspaces.

---

## Checklist

### ProjectPaths struct

- [ ] Create `src/paths/mod.rs` with a `ProjectPaths` struct as the single source of truth for all path construction
- [ ] Implement core path methods: `centy()`, `manifest()`, `config()`, `project_metadata()`, `users()`, `organization()`, `local_config()`
- [ ] Add sub-structs: `IssuePaths` (`dir()`, `content()`, `metadata()`, `assets()`, `links()`), `PrPaths`, `FeaturesPaths`, `TemplatesPaths`
- [ ] Add comprehensive unit tests for all path methods
- [ ] Migrate all modules to use `ProjectPaths` (priority order: `issue/crud.rs`, `pr/crud.rs`, `docs/crud.rs`, `config/mod.rs`, `workspace/storage.rs`, `link/storage.rs`, `manifest/mod.rs`, `reconciliation/*.rs`, `user/storage.rs`, etc.)
- [ ] Deprecate legacy helpers (`get_centy_path()`, `get_manifest_path()`, etc.) and remove after migration

### Unique project name enforcement

- [ ] On `SetProjectOrganization` (and any other org-assignment flow), validate that no other project in the target org shares the same directory name
- [ ] Return a clear error if the name is already taken, prompting the user to rename the project before assigning
- [ ] Add tests covering the uniqueness constraint

### Workspace folder naming

- [ ] Update `generate_workspace_path()` in `src/workspace/create.rs` to accept `project_name` and `issue_display_num` instead of `issue_id`
- [ ] Use naming scheme: `{project_name}-issue-{display_num}-{date}` (e.g., `my-app-issue-42-20231224`)
- [ ] Sanitize project name: lowercase, replace spaces/special chars with hyphens, truncate to ~30 chars, handle non-ASCII
- [ ] Extract project name from `source_project_path` (last directory component) or from `.centy/project.yaml`
- [ ] Guard against OS path-length limits (especially Windows 260-char default)
- [ ] Update `create_temp_workspace()` callers and all related tests

## Proposed `ProjectPaths` API

```rust
pub struct ProjectPaths {
    root: PathBuf,
}

impl ProjectPaths {
    pub fn new(project_path: impl AsRef<Path>) -> Self;

    pub fn centy(&self) -> PathBuf;
    pub fn manifest(&self) -> PathBuf;
    pub fn config(&self) -> PathBuf;
    pub fn project_metadata(&self) -> PathBuf;

    pub fn issues(&self) -> PathBuf;
    pub fn issue(&self, id: &str) -> IssuePaths;

    pub fn prs(&self) -> PathBuf;
    pub fn pr(&self, id: &str) -> PrPaths;

    pub fn docs(&self) -> PathBuf;
    pub fn doc(&self, slug: &str) -> PathBuf;

    pub fn features(&self) -> FeaturesPaths;
    pub fn templates(&self) -> TemplatesPaths;
    pub fn shared_assets(&self) -> PathBuf;

    pub fn users(&self) -> PathBuf;
    pub fn organization(&self) -> PathBuf;
    pub fn local_config(&self) -> PathBuf;
}

pub struct IssuePaths { base: PathBuf }

impl IssuePaths {
    pub fn dir(&self) -> &Path;
    pub fn content(&self) -> PathBuf;
    pub fn metadata(&self) -> PathBuf;
    pub fn assets(&self) -> PathBuf;
    pub fn links(&self) -> PathBuf;
}
```
