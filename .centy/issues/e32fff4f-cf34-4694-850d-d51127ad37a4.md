# Add dedicated command to list available daemon versions

Users should have an easy way to see all available daemon versions they can migrate to.

## Current State

* `centy info` shows available versions in daemon info output
* `centy version` shows project/daemon comparison but not available versions
* No dedicated way to list versions with their migration descriptions

## Expected Behavior

### Option A: Enhance existing command

````bash
$ centy version --list
Available versions:
  0.0.0 - Unversioned (base)
  0.1.0 - Establish version tracking system ← current
  
Your project: 0.1.0
Daemon: 0.1.0
Status: Up to date
````

### Option B: New subcommand

````bash
$ centy version list
Available versions:
  * 0.0.0 - Base version
  * 0.1.0 - Establish version tracking system (current)
````

## Implementation Notes

* Use existing `GetDaemonInfo` RPC which returns `available_versions`
* Could extend to show migration descriptions (requires new RPC or extending DaemonInfo)
* Consider marking current project version in the list

## Files

* `centy-cli/src/commands/version.ts` - Add –list flag
* OR create `centy-cli/src/commands/version/list.ts` - New subcommand
