# Generate .mcp.json during init command

When running the `init` command, ensure `.mcp.json` in the project root has the centy MCP server configured.

This file enables Claude Code (and other MCP clients) to discover and use the centy MCP server for the project without any manual setup.

## Behavior

The init command should **never blindly overwrite** `.mcp.json`. Instead:

1. **File does not exist** → create it with the centy entry
2. **File exists, `mcpServers.centy` absent** → parse and inject only the `centy` key, preserving all other content
3. **File exists, `mcpServers.centy` already present** → no-op (respect user's existing config)
4. **File exists but invalid JSON** → abort with a clear error message, do not modify the file

## Expected centy entry

```json
{
  "mcpServers": {
    "centy": {
      "command": "npx",
      "args": ["-y", "centy-mcp"]
    }
  }
}
```

## Notes

- The file should be created/updated alongside the other project init artifacts
- Preserve formatting and other keys when merging into an existing file
