# Truncate log file to prevent unbounded growth

The daemon log file can grow indefinitely over time. We should implement log truncation/rotation to keep the log file size manageable.\n\nOptions to consider:\n- Cap the log file at a maximum size and truncate/rotate when exceeded\n- Keep only the last N lines\n- Use a rolling log strategy (e.g. keep last N MB, rotate to .1 backup)\n\nThis prevents disk space issues for long-running daemon instances.
