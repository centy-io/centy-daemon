# Replace custom installer with cargo install

## Summary

Replace the custom `centy-installer` binary with `cargo install centy-daemon`.

## Motivation

The current installer is a full custom tool in the `installer/` repo: a shell script bootstraps a Rust binary that downloads, extracts, and places the daemon at `~/.centy/bin/centy-daemon`. This is significant maintenance overhead. Since the daemon is a Rust project, `cargo install` is the idiomatic distribution method.

## Changes needed

- Publish `centy-daemon` to crates.io
- Ensure the binary name in `Cargo.toml` matches (`centy-daemon`)
- Update CI/CD to publish to crates.io on release (alongside or instead of GitHub releases)
- Update any documentation referencing `~/.centy/bin/centy-daemon`

## Notes

- `cargo install` places the binary in `~/.cargo/bin/`, not `~/.centy/bin/`
- The CLI already has a PATH fallback in `get-search-paths.ts` so it will find the binary without code changes
- Pre-release installs: `cargo install centy-daemon --version x.y.z-pre.N` instead of the `--pre` flag
