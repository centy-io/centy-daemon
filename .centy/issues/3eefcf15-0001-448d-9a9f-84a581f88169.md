# centy-mcp version falls behind centy-daemon after daemon updates

## Problem

When `centy-daemon` is updated independently of the `centy-mcp` npm package (e.g. via `centy install`), the MCP binary at `~/.centy/bin/centy-mcp` can fall behind and become incompatible. The Claude Code MCP connection fails silently with "Failed to reconnect to centy.".

Observed: daemon at 0.10.5, installed centy-mcp at 0.9.2, npm latest at 0.12.0.

## Impact

Claude Code loses access to all Centy MCP tools until the user manually updates.

## Possible solutions

- Have `centy install` (or the daemon update path) also update `centy-mcp` to a matching version
- Have the MCP server emit a clear error or warning when its version is incompatible with the running daemon
- Add a version compatibility check on MCP startup that logs a human-readable message before failing
