# Implement item type registry scanning .centy/\*/config.yaml

Parent: #135

Implement the in-memory item type registry that the daemon builds at startup by scanning .centy/\*/config.yaml files.

## Scope

* Scan .centy/ direct subdirectories for config.yaml files
* Parse each config.yaml into the schema struct
* Build a HashMap\<String, ItemTypeConfig> registry keyed by folder name
* Validate no duplicate type names
* Skip directories without config.yaml (e.g., assets, templates)
* Log discovered types at startup
* Error handling: malformed config.yaml should log error and skip type, not crash daemon
* Unit tests with temp directories containing valid/invalid configs
