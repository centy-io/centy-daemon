# Replace custom git subprocess calls with git2 crate

## Context

\`src/common/git/\` contains several functions that invoke the git CLI via \`std::process::Command\` subprocess calls:

* \`branch.rs\`: \`detect_current_branch\`, \`validate_branch_exists\`, \`is_git_repository\`, \`get_default_branch\`
* \`git_remote.rs\`: \`get_remote_origin_url\`
* \`worktree.rs\`: \`create_worktree\`, \`remove_worktree\`, \`prune_worktrees\` (already marked deprecated)

## Problem

Shell-out to git is fragile: it relies on git being in PATH, is slower due to process spawning, and requires string parsing of output. The \`git2\` crate provides safe Rust bindings to libgit2 that handle all these operations natively.

## Proposed Change

1. Add \`git2 = “0.19”\` to \`Cargo.toml\`
1. Replace subprocess-based implementations in \`src/common/git/\` with \`git2\` API calls
1. The deprecated worktree functions can be deleted if they have no active callers

## Files

* \`src/common/git/branch.rs\`
* \`src/common/git/git_remote.rs\`
* \`src/common/git/worktree.rs\` (or delete)
