# Split link/storage/io.rs — separate file operations from serialization

## Problem

`src/link/storage/io.rs` (122 lines) mixes:
- File read/write operations
- Serialization format handling
- Link validation rules

## Proposal

- `storage/io.rs` — file operations only
- `storage/serialization.rs` — serialization / deserialization logic
- `storage/validation.rs` — link validation rules

Keeps each layer independently testable and easier to swap out (e.g. changing the storage format).
