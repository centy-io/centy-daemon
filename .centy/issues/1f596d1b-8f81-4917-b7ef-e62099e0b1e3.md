# Org repo path resolution and CreateItem with org_wide flag

First implementation step for the org-wide `.centy` repo feature (spec: `org-wide-centy-repo`). Covers auto-tracking the org repo on init, the registry-based discovery algorithm, and writing items into the org repo via a new `org_wide` flag on `CreateItem`.

## Auto-Tracking on Init

When `centy init` is run inside a directory named `.centy`, the daemon automatically recognizes it as the org-wide repo for that organization. No config field or manual designation is needed — the folder name is the signal.

The org repo is tracked like any other project in the global registry, but is queryable by its special role: a tracked project whose path ends in `/.centy`.

## Org Repo Discovery

Implement a per-request helper that, given a `project_path`, returns the org repo's data root (`Option<&TrackedProject>`):

1. Determine the current project's organization slug (from its registered org).
2. Scan tracked projects for one in the same org whose path ends in `/.centy`.
3. Return that project's path as the org repo data root, or `None` if not found.

No caching. No config fields. No filesystem heuristics.

## Proto Changes

Add to `CreateItemRequest`:

```protobuf
bool org_wide = N;
```

## Daemon Logic: `CreateItem` with `org_wide: true`

1. Resolve the org repo via the discovery helper above.
2. If no org repo is tracked, return a clear error.
3. Derive the originating project's slug from `project_path`.
4. Write the item to `<org_repo_path>/issues/<id>/` (the org repo root is the data root — no nested `.centy/` subfolder inside it).
5. Set `projects: [<slug>]` in the item's metadata.

## Acceptance Criteria

- [ ] `centy init` in a folder named `.centy` registers it as a normal tracked project (no special flag needed — discovery relies on path convention)
- [ ] Discovery helper scans tracked projects to find the org repo by org slug + path suffix `/.centy`
- [ ] `CreateItemRequest` has `org_wide: bool` in proto
- [ ] `CreateItem` with `org_wide: true` writes to the org repo data root and sets `projects: [<slug>]` in metadata
- [ ] `CreateItem` with `org_wide: true` when no org repo is tracked returns a clear error
- [ ] Unit tests cover discovery (found, not found) and the CreateItem org_wide write path
