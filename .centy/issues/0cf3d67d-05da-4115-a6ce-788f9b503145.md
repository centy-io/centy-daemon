# Replace merge_json_content with a library

## Background

The `merge_json_content` function in `src/reconciliation/managed_files_merge.rs` is a hand-rolled JSON merge implementation that:
- Unions array values for `words` and `ignorePaths` keys (deduplicates + sorts case-insensitively)
- Overrides scalar fields (`version`, `language`) from the template
- Preserves user-added keys not present in the template

## Problem

This is custom logic that a well-maintained crate could handle more robustly and with less maintenance burden.

## Task

Evaluate and adopt a suitable Rust crate (e.g. `json-patch`, `serde_merge`, or similar) to replace the hand-rolled logic. The replacement should:
- Preserve current behavior exactly (existing tests in `managed_files_json_merge.rs` and `managed_files_json_sorting.rs` must pass)
- Remove or simplify `merge_json_content` in `managed_files_merge.rs`
- Update `Cargo.toml` with the new dependency
