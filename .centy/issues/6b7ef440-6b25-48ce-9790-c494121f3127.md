# Replace custom git remote URL parsing with git-url-parse crate

## Context

\`src/common/remote.rs\` (~60 lines) contains a custom implementation to parse git remote URLs in SSH and HTTPS formats, extracting host, org, and repo name.

## Problem

The crate [\`git-url-parse\`](https://crates.io/crates/git-url-parse) (or \`gix-url\`) handles all standard git URL formats (SSH, HTTPS, HTTP, SCP-style) and extracts the same fields. The custom implementation may miss edge cases like port numbers, subgroups, or unusual URL formats.

## Proposed Change

1. Add \`git-url-parse = “0.4”\` (or equivalent) to \`Cargo.toml\`
1. Replace \`parse_remote_url\` in \`src/common/remote.rs\` with a wrapper around the crate’s parser
1. Remove the custom SSH and HTTPS parsing functions

* \`src/common/remote.rs\` — custom URL parser to replace
