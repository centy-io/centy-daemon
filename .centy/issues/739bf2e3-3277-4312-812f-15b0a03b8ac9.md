# Add projects field to item metadata

Add a `projects` field to both item frontmatter and `CreateItemRequest` (spec: `org-wide-centy-repo`).

## CreateItemRequest

Add `projects: repeated string` to `CreateItemRequest`. This field serves two purposes:

1. **Routing**: `len(projects) > 1` signals the daemon to write the item to the org repo (handled in #384).
2. **Association**: stored as-is in the item's frontmatter.

## Frontmatter Change

Add `projects` to item frontmatter — a list of project slugs:

```markdown
---
projects: ["frontend", "backend"]
---
```

The field is optional and defaults to an empty list. For regular per-project items it stays empty or has a single entry. For org-wide items it contains two or more project slugs.

## Changes Required

- **Storage**: Add `projects: Vec<String>` to the frontmatter struct
- **Proto**: `repeated string projects` already added to both `GenericItemMetadata` and `CreateItemRequest`
- **Serialization**: Read/write `projects` in the markdown frontmatter; default to `[]` when the field is absent (backwards-compatible)

## Acceptance Criteria

- [ ] `projects` field exists on the frontmatter struct as `Vec<String>`
- [ ] Serialized to/from markdown frontmatter; missing field deserializes as empty list
- [ ] `CreateItemRequest.projects` is wired through to the stored frontmatter
- [ ] Existing items without the field continue to load correctly
