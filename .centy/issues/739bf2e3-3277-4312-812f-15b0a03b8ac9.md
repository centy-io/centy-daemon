# Add projects field to item metadata

Add a `projects` field to item metadata to record which projects an item is associated with. This is a prerequisite for org-wide items (spec: `org-wide-centy-repo`), where the field identifies which project(s) an org-level item is relevant to.

## Metadata Change

Add `projects` to item metadata — a list of project slugs:

```json
{
  "projects": ["frontend", "backend"]
}
```

The field is optional and defaults to an empty list. For regular per-project items it stays empty. For org-wide items it is populated at creation time with the originating project's slug.

## Changes Required

- **Storage**: Add `projects: Vec<String>` to the metadata struct (alongside existing fields like `status`, `priority`, `tags`)
- **Proto**: Expose `projects` in the item message so clients can read it
- **Serialization**: Read/write `projects` in `metadata.json`; default to `[]` when the field is absent (backwards-compatible)

## Acceptance Criteria

- [ ] `projects` field exists on the metadata struct as `Vec<String>`
- [ ] Serialized to/from `metadata.json`; missing field deserializes as empty list
- [ ] Exposed in the proto item message
- [ ] Existing items without the field continue to load correctly
