# UpdateItem / DeleteItem: route writes to org repo

Extend `UpdateItem` and `DeleteItem` to transparently route writes to the org repo when the target item lives there (spec: `org-wide-centy-repo`). Depends on #384 (org repo discovery).

## Daemon Logic

Both handlers currently resolve the item path relative to the project's own `.centy/`. They need an additional step:

1. Attempt to locate the item in the project's own `.centy/`.
2. If not found there, check the org repo (via the discovery helper from #384).
3. Route the write (update or delete) to whichever location the item was found in.
4. If not found in either, return the usual not-found error.

The caller does not need to know or specify where the item is stored — routing is fully transparent.

## Acceptance Criteria

- [ ] `UpdateItem` on an org-wide item writes to the org repo, not the project's `.centy/`
- [ ] `DeleteItem` on an org-wide item deletes from the org repo
- [ ] If the item is not in the org repo either, the usual not-found error is returned
- [ ] Project-local items are unaffected — routing is unchanged when item is found in project
- [ ] When no org repo is tracked, behavior is unchanged (not-found error if item absent from project)
- [ ] Tests cover: update/delete project item, update/delete org-wide item, item not found in either, no org repo tracked
