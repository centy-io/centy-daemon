# Split workspace_temp handler — separate creation, editor, and hooks phases

## Problem

`src/server/handlers/workspace_temp/handler.rs` (100 lines) handles multiple distinct phases in a single file:
- Workspace creation orchestration
- Editor invocation
- Status update hooks
- Error response formatting
- Project initialization checks

## Proposal

- `handler.rs` — thin orchestration
- `operations/create.rs` — workspace creation logic
- `operations/editor.rs` — editor invocation
- `hooks.rs` — status update hook logic
