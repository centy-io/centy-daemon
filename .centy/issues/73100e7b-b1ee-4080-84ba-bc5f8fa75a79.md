# Define config.yaml schema for item types

Parent: #135

Define the config.yaml schema that describes an item type. This is the contract that all item types (built-in and custom) follow.

## Scope

- Define the Rust struct for parsing config.yaml (serde + serde_yaml)
- Fields: name, icon, features (displayNumber, status, priority, softDelete, assets, orgSync, move, duplicate), statuses, defaultStatus, priorityLevels, customFields, template
- Validation: required fields, valid status names, priority levels > 0
- Unit tests for parsing valid and invalid config.yaml files
- Create example config.yaml files for issue and doc types

## Example config.yaml

```yaml
name: Issue
icon: clipboard
features:
  displayNumber: true
  status: true
  priority: true
  softDelete: true
  assets: true
  orgSync: true
  move: true
  duplicate: true
statuses:
  - open
  - in-progress
  - closed
defaultStatus: open
priorityLevels: 3
customFields: []
template: template.md
```
