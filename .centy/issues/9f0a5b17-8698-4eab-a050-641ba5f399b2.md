# Version gRPC API with package centy.v1

## Summary

Adopt the standard gRPC/Google API versioning convention by changing the proto package from `centy` to `centy.v1`.

## Changes Required

1. Restructure proto directory: `proto/centy.proto` → `proto/centy/v1/centy.proto`
1. Update package declaration: `package centy;` → `package centy.v1;`
1. Update all generated code imports to reflect the new package path
1. Update all server/client code referencing the old service path (`/centy.CentyDaemon/...` → `/centy.v1.CentyDaemon/...`)
1. Update proto compilation scripts/config (buf, protoc, etc.)
1. Update e2e tests to use the new service paths

## Why

* Follows Google API / Buf standard convention for gRPC versioning
* Enables running v1 and v2 side-by-side on the same server in the future
* Breaking changes can go in v2; non-breaking additions stay in v1
* Cheap to do now, expensive to do later as more clients depend on the wire format

## Notes

* This is a breaking change for all existing clients — coordinate CLI + daemon release together
* Wire format changes from `/centy.CentyDaemon/RPC` to `/centy.v1.CentyDaemon/RPC`
