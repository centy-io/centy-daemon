# Remove PR entity type

Remove the entire PR (pull request) subsystem from the daemon. PRs will not be a built-in entity type.

## What to remove

### Source code

* `src/item/entities/pr/` - entire PR entity module (create, crud, git, metadata, status, id, reconcile, remote)
* `src/server/handlers/pr_*.rs` - all PR handler files (pr_create, pr_read, pr_list, pr_update, pr_delete, pr_soft_delete, pr_restore, pr_by_uuid)
* All PR-related exports from `src/lib.rs`
* PR references in `src/server/` (router registration, handler mod declarations)
* `HookItemType::Pr` variant from hooks config

### Proto definitions

* All PR RPC methods from `proto/centy/v1/centy.proto` (CreatePr, GetPr, GetPrByDisplayNumber, GetPrsByUuid, ListPrs, UpdatePr, DeletePr, SoftDeletePr, RestorePr, GetNextPrNumber)
* All PR-related message types (requests, responses, PrMetadata, PrWithProject)

### Tests

* `tests/pr_test.rs` - PR unit tests
* `e2e/prs.e2e.spec.ts` - PR e2e tests

### Data

* `.centy/prs/` directory references in initialization code
* PR-related default config (statuses, etc.)
