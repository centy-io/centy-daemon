# Unify move logic for all item types (static and custom)

The move operation exists only for issues and docs but not in the generic layer. We should add it to the generic layer and route all types through it.

## Current state

* **Generic**: No `generic_move()` exists
* **Issues**: `move_issue()` in `src/item/entities/issue/crud.rs` — moves to another project, reassigns display number
* **Docs**: `move_doc()` in `src/item/entities/doc/crud.rs` — moves to another project with optional slug rename

gRPC handlers:

* No unified `item_move.rs` exists
* `issue_move.rs` → calls `issue::move_issue()`
* `doc_move.rs` → calls `doc::move_doc()`

## Goal

1. Create `generic_move()` in `src/item/generic/storage.rs`
1. Remove `move_issue()` and `move_doc()` type-specific implementations
1. Create unified `MoveItem` gRPC handler in `item_move.rs`
1. `MoveIssue` and `MoveDoc` handlers should route through the unified move layer
1. Display number reassignment and slug rename should be handled generically based on identifier type (UUID vs slug)

## Files involved

* `src/item/generic/storage.rs` — add `generic_move()` (new)
* `src/item/entities/issue/crud.rs` — `move_issue()` to remove
* `src/item/entities/doc/crud.rs` — `move_doc()` to remove
* `src/server/handlers/item_move.rs` — new unified handler
* `src/server/handlers/issue_move.rs` — to redirect to unified
* `src/server/handlers/doc_move.rs` — to redirect to unified
