# Split init/mcp_json.rs — separate file I/O from MCP config logic

## Problem

`src/server/handlers/init/mcp_json.rs` (163 lines, ~half tests) mixes:
- JSON file parsing and validation
- File I/O operations
- MCP-specific configuration logic

## Proposal

- `mcp_json.rs` — JSON serialization / deserialization logic
- `mcp_io.rs` — file read/write operations

This makes the I/O layer independently testable with stubs, and keeps the config schema logic focused.
