# worktree open rejects UUID-format Centy issue numbers

## Bug

When running `worktree open centy:<uuid>`, the daemon errors with:

```
Error: Invalid Centy issue number: "6f4853a9-3d82-4013-b909-c2d637f44541" — expected a positive integer
[worker_pool] `worktree open centy:6f4853a9-3d82-4013-b909-c2d637f44541` exited with 1
```

## Root Cause

The `worktree open` command parses the Centy issue reference and validates it expects a positive integer, but the identifier passed is a UUID (the item's internal ID rather than its display number).

## Expected Behavior

Either:
1. Accept UUIDs in addition to display numbers when resolving Centy issue references, or
2. Return a clearer error indicating the display number (e.g. `#42`) should be used instead of the internal UUID.
