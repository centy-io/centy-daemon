# Custom templates per item type

Templates define the starting point for every new item in Centy. When a user creates an issue or a doc, the template determines what the initial content looks like â€” the sections, the prompts, the structure. But right now, templates are generic. There is no way to have different templates for different kinds of work within the same item type, and there is no way to tailor templates to the specific needs of a custom item type.

## Design Decision: Templates live inside the item type folder

With the config.yaml approach (see #135), each item type is self-contained in its .centy/<type>/ folder. Templates follow the same principle:

```
.centy/
  issues/
    config.yaml           # references template(s)
    templates/
      default.md          # default issue template
      bug-report.md       # alternative template for bug reports
      feature-request.md  # alternative template for feature requests
    0001.md
  features/
    config.yaml
    templates/
      default.md          # default feature template
    feat-login.md
```

The config.yaml for each type specifies the default template and lists available templates:

```yaml
template: templates/default.md
templates:
  - name: default
    path: templates/default.md
  - name: bug-report
    path: templates/bug-report.md
  - name: feature-request
    path: templates/feature-request.md
```

When `centy init` creates the built-in issue and doc types, it also creates their default template files.

## Key aspects

* Each item type (built-in and custom) supports multiple named templates inside its own folder
* Users select a template when creating a new item, with the config.yaml default as fallback
* Templates are stored as files inside the item type folder for easy editing and version control
* Template files support variables and placeholders (date, author, item number)
* It should be possible to set a default template per item type in config.yaml
* Teams can share templates across projects through organization-level configuration
* Creating a new template is just creating a new file in the templates/ subfolder
