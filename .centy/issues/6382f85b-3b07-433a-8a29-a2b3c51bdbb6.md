# Migrate link storage to per-link markdown files with frontmatter

## Summary

Change how links are stored: instead of per-entity JSON files (`.centy/{type}s/links/{entity_id}/links.json`), each link becomes its own markdown file at `.centy/links/{uuid}.md` with all data in YAML frontmatter — making links consistent with how generic items are stored.

## New storage format

```
.centy/links/{uuid}.md
```

Frontmatter fields:
- `sourceId` — source entity UUID
- `sourceType` — singular item type (e.g. `issue`)
- `targetId` — target entity UUID
- `targetType` — singular item type
- `linkType` — relationship type (e.g. `blocks`)
- `createdAt` / `updatedAt` — timestamps from mdstore

## Key decisions

- **One file per relationship** (no separate inverse file; inverse synthesized at query time)
- **No display numbers**
- **Drop old format** (no backward compat migration)
- Use `mdstore::create/delete/list` with `TypeFeatures::default()` (all features off)

## Changes required

- New `LinkRecord` struct (full both-sides record) + `source_link()` / `target_link(inverse)` view methods
- Rewrite `storage/io.rs` to use mdstore on `.centy/links/`
- Delete `storage/links_file.rs` and `LinksFile` type
- Rewrite `crud_fns/create.rs` — one file, synthesize both response views
- Rewrite `crud_fns/delete.rs` — scan+filter files by source/target, delete matched
- Rewrite `crud_read.rs` — `list_links` scans all files, filters by entity, synthesizes inverse for target matches; add `custom_types` param
- Update `link/mod.rs` exports (remove `LinksFile`, `read_links`, `write_links`)
- Update `server/handlers/link_read.rs` — fetch `custom_types` before calling `list_links`
- Rewrite `links_file_*` tests for new storage model
