# Split item_list filters.rs into per-filter-type modules

## Problem

`src/server/handlers/item_list/filters.rs` (158 lines) handles 5 independent filter types in one file:
- Custom field parsing
- Status conditions
- Priority conditions (with `$eq`, `$lt`, `$lte`, `$gt`, `$gte` operators)
- Tag union/intersection logic
- MQL query parsing entry point

Each filter type is independent and could be tested in isolation.

## Proposal

Split into a `filters/` submodule:
- `filters/mod.rs` — main filter builder / entry point
- `filters/custom_fields.rs`
- `filters/status.rs`
- `filters/priority.rs`
- `filters/tags.rs`
