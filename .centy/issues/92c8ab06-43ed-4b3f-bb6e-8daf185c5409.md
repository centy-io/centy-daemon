# Add test targets to Makefile and use them in pre-push hook

## Problem

The root `Makefile` only contains `build` targets. The `.husky/pre-push` hook runs tests and coverage checks by calling `cargo` directly:

```sh
CENTY_HOME="$CENTY_HOME_TEST" cargo test --all-targets
CENTY_HOME="$CENTY_HOME_COV" cargo llvm-cov --all-targets --ignore-filename-regex "$IGNORE_REGEX" --fail-under-lines 100
```

This is inconsistent with how the build step is invoked (`make build`) and makes it harder to run tests locally in the same way the hook does.

## Proposed fix

Add `test` and `coverage` targets to the root `Makefile`, then update `.husky/pre-push` to call them.

### Makefile additions

```makefile
.PHONY: test coverage

test:
	cargo test --all-targets

coverage:
	cargo llvm-cov --all-targets --ignore-filename-regex "$(IGNORE_REGEX)" --fail-under-lines 100
```

The `IGNORE_REGEX` value can either be inlined in the Makefile or kept as an environment variable passed through from the hook.

### pre-push hook changes

Replace the inline `cargo test` and `cargo llvm-cov` calls with:

```sh
echo "[5/9] Running tests..."
CENTY_HOME="$CENTY_HOME_TEST" make test

echo "[6/9] Checking test coverage..."
CENTY_HOME="$CENTY_HOME_COV" make coverage
```

## Acceptance criteria

- `make test` runs the full test suite the same way the pre-push hook does
- `make coverage` runs llvm-cov with the same ignore regex and line threshold
- The pre-push hook delegates to `make test` and `make coverage` instead of calling `cargo` directly
