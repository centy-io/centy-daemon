# Remove agent/LLM logic from the daemon

## Overview

The daemon currently contains two categories of code that do not belong here: an `llm` module for LLM/agent orchestration, and server handlers that launch AI agents inside terminal windows. Both must be removed. The daemon's responsibility ends at **opening** editors and terminals ‚Äî it must never spawn agents, execute commands inside them, or manage LLM work sessions.

---

## What to Remove

### 1. The `src/llm/` module (entire directory)

* `src/llm/mod.rs` ‚Äî module exports
* `src/llm/work.rs` ‚Äî work session tracking (`llm-work.json`)
* `src/llm/agent.rs` ‚Äî agent spawning logic
* `src/llm/prompt.rs` ‚Äî prompt building
* `src/llm/config.rs` ‚Äî LLM configuration
* `src/llm/generate.rs` ‚Äî title generation

### 2. Agent orchestration handlers

* `src/server/handlers/workspace_agent.rs` ‚Äî `open_agent_in_terminal()` handler
* `src/server/handlers/workspace_agent_open.rs` ‚Äî `open_workspace_and_terminal()` orchestration

### 3. Agent launching logic in workspace module

* `src/workspace/terminal.rs` ‚Äî remove `open_terminal_with_agent()` function; keep `open_terminal()` and other plain terminal-opening utilities
* `src/workspace/mod.rs` ‚Äî remove re-export of `open_terminal_with_agent`; also clean up any `llm` module imports

### 4. gRPC API (agent-specific endpoints)

* `OpenAgentInTerminal` RPC endpoint in `proto/centy/v1/centy.proto`
* `OpenAgentInTerminalRequest` and `OpenAgentInTerminalResponse` message types
* `WorkspaceMode` enum (if only used by the agent flow)

### 5. gRPC trait implementation

* `src/server/trait_impl.rs` ‚Äî `open_agent_in_terminal()` impl

### 6. References to update / additional cleanup

* `src/lib.rs` ‚Äî remove `llm` module import
* `src/server/mod.rs` ‚Äî remove llm-related imports
* `src/item/entities/issue/create.rs` ‚Äî remove llm usage
* `src/config/mod.rs` (`WorkspaceConfig`) ‚Äî remove `workspace.update_status_on_open` if it only serves the agent flow
* Remove `llm_work()` path from `ProjectPaths` struct (see issue #81)
* Remove any `llm-work.json` file handling
* Remove `.centy/docs/agent-manager.md` ‚Äî agent manager design doc
* Clean up related dependencies in `Cargo.toml`

---

## What to Keep

* `open_terminal()`, `is_terminal_available()`, `open_platform_terminal()` ‚Äî plain terminal/editor opening without executing anything inside
* Any RPC or handler that only opens an editor window (no agent execution)

---

## Guiding Principle

The daemon can open editor/terminal windows but must never run anything inside them. All logic that executes commands or spawns agents belongs in the client (CLI/desktop app), not the daemon. The daemon focuses on project/issue management and plain editor/terminal launching.

---

## Notes

* Ensure no other handlers or CLI commands depend on removed code before deleting
* After cleanup, verify that the remaining terminal-opening code does not pass any commands or agent arguments to the terminal
