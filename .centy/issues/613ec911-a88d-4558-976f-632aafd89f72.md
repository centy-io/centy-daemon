# Replace custom duration string parsing with humantime crate

## Context

\`src/cleanup/parse.rs\` contains a custom \`parse_retention_duration\` function (~25 lines) that manually parses duration strings like \`"30d"\`, \`"24h"\` into \`chrono::Duration\`.

## Problem

This is reinventing functionality already provided by the [\`humantime\`](https://crates.io/crates/humantime) crate. The custom parser only supports \`d\` (days) and \`h\` (hours) suffixes, while humantime supports a much richer format.

## Proposed Change

1. Add \`humantime = "2"\` to \`Cargo.toml\`
2. Replace \`parse_retention_duration\` in \`src/cleanup/parse.rs\` with a thin wrapper over \`humantime::parse_duration\` that converts \`std::time::Duration\` to \`chrono::Duration\`
3. Remove the custom parsing logic

## Files

- \`src/cleanup/parse.rs\` — the custom parser to replace
