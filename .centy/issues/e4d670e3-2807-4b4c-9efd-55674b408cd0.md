# Remove all ignore regexes from the pre-push coverage check

The pre-push hook's coverage step uses a large `IGNORE_REGEX` passed to `cargo llvm-cov --ignore-filename-regex` to exclude dozens of source files from the 100% line-coverage requirement. The goal is to achieve genuine 100% coverage so none of those exclusions are needed.

**What needs to be done:**
- Write tests for every module currently listed in `IGNORE_REGEX` inside `.cargo-husky/hooks/pre-push` (the canonical source; `.git/hooks/pre-push` is the installed copy).
- Once every module is covered, remove the `IGNORE_REGEX` variable and the `--ignore-filename-regex` flag from the `cargo llvm-cov` invocation in both files, leaving the step as simply:
  ```sh
  CENTY_HOME="$CENTY_HOME_COV" cargo llvm-cov --all-targets --fail-under-lines 100
  ```
- Confirm the pre-push hook passes end-to-end with no exclusions.
